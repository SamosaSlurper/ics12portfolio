Tutorial: Cycle Sort

The Basics

Cycle sort is an in-place sorting algorithm that sorts an array by minimizing the number of memory writes. Elements are shifted constantly and replace the position of other elements in a circular fashion until all the elements are in their correct positions.

Complexity

  • Time Complexity: O(N2) for worst, average, and best cases. This is because to find the correct position of an element, we basically have to count all elements smaller than it across the array.
  • Space Complexity: O(1). Since it's an in-place sorting algorithm, it doesn't need any extra space.

How It Works

The concept is pretty easy to grasp. You take an element, and count how many elements in the rest of the array are smaller than it. That count tells us the index where our element belongs at. We move it to that index, and kick out (or give the boot) to whichever element is already sitting there.

Then, we take that kicked-out element and do the same thing: find out how many elements are smaller than it to find its correct index, and move it there. We repeat this process until the initial element's index position is filled with the appropriate element. This completes one "cycle". A "cycle" is counted when an element reaching the index position of the initial value that was moved. It then checks whether the loop is in order or not. If it isn't the program will move to a different element that is unsorted and procede with the process until the array is fully sorted.

Important Note: If an element is already in the correct position from the start, nothing happens. We just move on to the next index and it does NOT does not count as a cycle. A cycle is only counted when an element is actually moved.

Syntax

1. Loop Through the Array

Starting Point

for (int start = 0; start < array.length - 1; start++) {
    int item = array[start];
    int pos = start;

2. Count Smaller Elements

Target Index

    for (int i = start + 1; i < array.length; i++) {
        if (array[i] < item) {
            pos++; // increment position
        }
    }

3. Handle Duplicates & Skip Correct Elements

Skip Duplicates

    if (pos == start) continue; // already sorted

    while (item == array[pos]) {
        pos++; // skip duplicate
    }

4. Swap & Complete the Cycle

Final Swap

    if (pos != start) {
        int temp = item;
        item = array[pos];
        array[pos] = temp; // swap
    }
} // repeat until you return to starting index

5. Count Cycles

Track Cycle Count

    cycles++; // count cycles
}

An Example

Let's look at an example array: [5, 2, 4, 1, 3]. We'll track the moves and count the cycles.

Cycle 1: Starting at index 0 (Value = 5)

  • Count elements smaller than 5. There are 4 of them (2, 4, 1, 3). So 5 goes to index 4.
  • Move 5 to index 4. Kick out the element there (which is 3).
  • Now we have to place 3. Count elements smaller than 3. There are 2 of them (2, 1). So 3 goes to index 2.
  • Move 3 to index 2. Kick out the element there (which is 4).
  • Now we have to place 4. Count elements smaller than 4. There are 3 of them (2, 1, 3). So 4 goes to index 3.
  • Move 4 to index 3. Kick out the element there (which is 1).
  • Now we have to place 1. Count elements smaller than 1. There are 0 of them. So 1 goes to index 0.
  • Since index 0 is our starting position, we put 1 there and Cycle 1 is complete!

Impact & Importance

The Impact on a Program

In terms of impact, it boils down to its efficiency in memory writes. As cycle sort minimizes the number of time you have to write to memory on the basis that if it's already in the right place, the element is written 0 times, it's the logical go-to solution when sorting data using EEPROMs (Electrically Erasable Programmable Read-Only Memory).

Common Mistakes & How to Avoid Them

The most common mistake would probably be a miscalculation with the counting. As you count the number of elements smaller than the value of the current element, many errors could occur, like miscounting the wrong number causing the target index to be incorrect setting off a chain reaction where data will get stuck and overwrite the incorrect data. As a key to avoid this from happening, it's important to loop the entire rest of array when counting so duplicate values can be handled appropraitely by avoiding them.

What if we didn't use this?

Cycle sort doesn't have many alternatives as it is one of a kind in a way. That being said though, without cycle sort we would probably rely on other sorting methods like Quick Sort or Merge Sort. While they can be faster in time complexity, the number of memory writes drastically increases. If this were to happen, certain hardwares and drives such as embedded systems would be unable to run for a longer period of time as they would wear out a lot faster.
